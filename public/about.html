The full story

I'm a B.Tech student specializing in AI & Data Science at MIT World Peace University, Pune. I got into AI not because it was trendy, but because I was frustrated with how many impressive demos never became real systems. So I made it my goal to close that gap myself.

My work spans three domains: LLM systems and RAG pipelines (NyayaAI, ResolvAI, BharatNiti), computer vision applications (Tulsi Disease Detection, FaceMark attendance), and full-stack AI products that bring both together (CyberRakshak, Probity.ai). I care about privacy-first architectures — if a system can run locally, it should.

Beyond code, I think about AI infrastructure and deployment. Building a model is the easy part. Making it reliable, fast, and maintainable under real-world conditions is where the interesting work happens. That's the problem space I'm most interested in.

How I think

Design philosophy

Production over polish

A beautiful model that doesn't deploy is a failed project. I build with the endpoint in mind from day one — API contracts, error handling, latency budgets, and real-world data distribution.

Privacy as a constraint, not an afterthought

ResolvAI and CyberRakshak both run entirely offline. If sensitive data is involved, it should never leave the machine. This shapes architecture decisions from the start.

Explainability is a feature

FaceMark uses classical CV intentionally — LBPH and Haar Cascade over a neural network — because the recognition logic is transparent and debuggable. Explainability isn't a regression; sometimes it's the right tool.
